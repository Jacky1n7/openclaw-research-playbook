# Context / Token 管理（会话变长时的收敛策略）

目标：在长会话/多轮工具调用中保持：
- 不跑偏（证据链清晰）
- 不刷屏（输出可读）
- 不爆上下文（及时收敛）

## 触发阈值（建议）

- 上下文占用 > **70%**：进入“收敛模式”
- > **85%**：强制“产物优先 + 停止发散”

## 收敛模式（>70%）

1) **先产物后聊天**
- 优先把关键内容写入：`artifacts/` 或 `memory/YYYY-MM-DD.md`
- 聊天只发：结论 + 证据指向（文件路径/链接）

2) **Redundancy check 更严格**
- 相似工具调用直接复用结果
- 网络搜索/状态检查设 TTL（10–30min）

3) **摘要替代全文**
- 只保留：5–10 条要点 + 1 个 TODO
- 把长内容放文件，不在对话里堆

## 强制模式（>85%）

- 立即生成“会话摘要块”并写入 `memory/YYYY-MM-DD.md`
- 停止继续扩展问题范围，要求人类选择下一步（2–3 个选项）

## Cron 学习任务的固定输出格式

每次 Moltbook 学习（08:00/13:00/18:00）必须用这个格式：

- A（工作流）洞见：<= 3 条
- B（交互）洞见：<= 3 条
- C（安全）洞见：<= 3 条
- Ops（运维）洞见：<= 3 条
- 1 个可执行 TODO
- 1 段拟发帖/评论（100–200 字）

> 若上下文紧张：只输出 A/B/C/Ops 各 1 条 + TODO。
